## ___Wireless Topology Modes___
- __Ad hoc mode__
	- two devices connects wirelessly in a peer-to-peer
	- No using APs or wireless router
	- Independent basic service set (IBSS)
- __Infrastructure mode__
	- when clients interconnect via a wireless device
	- Uses the wired distribution system (Ethernet)

## ___BSS and ESS___
- ___BSS (Basic Service Set)___
	- Single AP interconnecting all associated wireless clients.
	- MAC addresses of a AP is used to uniquely identify each BSS (BSSID - Basic Service Set Identifier)
- ___ESS (Extended Service Set)___
	- Two or more BSSs connected trough a common __DS (distribution system)__.

## ___CSMA/CA___
- WLANs are half-duplex, shared media
- __Shared media__ - wireless clients can all transmit/receive on the same frequency
- __CSMA/CA(Carrier Sense Multiple Access/ Collision Avoidance )__ - method to determine _how_ and _when_ to send data on the network.
- ___Steps:___
	1. Listens to the channel to see if it's _idle_ (no other traffic).
	2. Sends a __request to send (RTS)__ message to the AP.
	3. Receives a __clear to send (CTS)__ message from the AP.
	4. If doesn't receive the message, waits random amount of time before restart the process.
	5. If receive, transmits the data.
	6. All transmissions are acknowledged.

## ___Passive/Active Discovery Mode___
- __Passive Mode__
	- Openly advertise its service periodically broadcast beacon frames.
	- Purpose of the beacon is to allow client to learn which networks and APs are in the area.
- __Active Mode__
	- Must know the name of the SSID.
	- Client broadcast a probe request frame (SSID, standard supported). 
	- APs with the SSID sends the probe response
