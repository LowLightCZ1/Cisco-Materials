## ___Steps for Router Failover___
- If router fail, protocol transitions the standby router to a new active role
__Steps of a router__
1. Stop sending _hello_ messages from the forwarding router
2. Assumes the new role
3. Devise see no disruption in service

## ___FHRP Options___
- __Hot Standby Router Protocol(HSRP)__
	- Cisco-proprietary FHRP
	- allow for transparent failover of a first-hop IPv4 device
	- high network availability
	- Used in group of routers 
- __HSRP for IPv6__
	- Same as HSRP, but in IPv6 environment
- __Virtual Router Redundanci Protocol v2(VRRPv2)__
	- non-proprietary protocol
	- dynamically assigns responsibility for virtual routers 
	- allows routers on multiaccess link use the same virtual IPv4 address
